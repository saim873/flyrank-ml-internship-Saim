## How I Will Add My Next Case Study

For every new project, I will reuse the same three-beat case-study structure:

1. **Problem** — Clearly explain the real problem, who it affects, and why it matters.
2. **What I Did** — Describe the tools, data, methodology, decisions, and validation used in the project.
3. **What Came of It** — Present the measured results, limitations, lessons learned, and the next recommended action.

After completing the case study, I will:

- Add the project files and documentation to GitHub.
- Write a clear README and reproducibility instructions.
- Create or update the public portfolio page.
- Test the project and links on desktop and mobile.
- Share the completed case study on LinkedIn.
